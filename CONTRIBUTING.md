# Contributing

## Changelog

Every pull request should update `CHANGELOG.md` under `[Unreleased]` with one
short line per user-visible change. Use the Keep a Changelog sections that fit:
`Added`, `Changed`, `Deprecated`, `Removed`, `Fixed`, or `Security`.

For verification-only PRs, add a `Changed` or `Fixed` note only when the PR
changes behavior. Pure reports do not need a changelog entry.
